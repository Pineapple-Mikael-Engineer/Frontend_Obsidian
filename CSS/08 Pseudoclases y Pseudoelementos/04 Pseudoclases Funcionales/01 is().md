# :is()

> [!definicion]
> `:is(selector, selector, …)` agrupa varios selectores en uno, evitando la repetición. Coincide si el elemento casa con **cualquiera** de los selectores de la lista. Simplifica enormemente los selectores largos y repetitivos.

```css
:is(h1, h2, h3) { margin-top: 2rem; }
/* = h1, h2, h3 { margin-top: 2rem; } */
```

## Evitar la repetición

> [!tip] Factoriza partes comunes de selectores
> El verdadero poder aparece con selectores **anidados** repetitivos. Sin `:is()`:
> ```css
> /* ❌ verboso y repetitivo */
> article h1, article h2, article h3,
> aside h1, aside h2, aside h3 { font-family: serif; }
> ```
> Con `:is()`:
> ```css
> /* ✅ conciso */
> :is(article, aside) :is(h1, h2, h3) { font-family: serif; }
> ```
> Las combinaciones se multiplican: `:is()` evita escribir el producto cartesiano a mano.

## La especificidad: la del más alto

> [!warning] :is() toma la especificidad mayor de su lista
> Un detalle importante: la [[01 Especificidad/index | especificidad]] de `:is()` es la del selector **más específico** de su lista, no la del que coincide:
> ```css
> :is(#id, .clase, p) { }   /* especificidad de #id (0,1,0,0) para TODOS */
> ```
> Aunque coincida por el `p`, el selector pesa como un `#id`. Esto puede dar sorpresas: un `:is()` con un `#id` dentro hace que toda la regla sea muy específica. Si quieres agrupar **sin** añadir especificidad, usa [[02 where() | `:where()`]].

## Tolerancia a errores

> [!info] :is() ignora los selectores inválidos
> Otra ventaja: si un selector dentro de `:is()` es **inválido** (no reconocido por el navegador), `:is()` **ignora** solo ese y mantiene los demás. En una lista de selectores normal (separada por comas), un selector inválido **invalida toda la regla**. `:is()` es más robusto:
> ```css
> :is(::selección-rara, p) { }   /* el p sigue funcionando aunque el otro sea inválido */
> ```

## Casos de uso

```css
/* Estilos de enlaces en varios contextos */
:is(nav, footer, .menu) a { color: inherit; }

/* Encabezados en cualquier sección */
:is(section, article) :is(h2, h3) { scroll-margin-top: 4rem; }

/* Estados combinados */
button:is(:hover, :focus-visible) { background: #b48ef0; }
```

## Buenas prácticas

> [!tip] Recomendaciones
> - Usa `:is()` para factorizar selectores repetitivos (sobre todo anidados).
> - Recuerda que su especificidad es la del **más alto** de la lista; cuidado con meter `#id`.
> - Si quieres agrupar sin añadir especificidad, usa `:where()`.
> - Aprovecha su tolerancia a selectores inválidos para usar características nuevas con respaldo.

## Errores comunes

> [!warning] Trampas
> - **Especificidad inesperada** por un `#id` o `.clase` dentro de `:is()`.
> - **Confundir `:is()` con `:where()`**: uno mantiene especificidad, el otro la anula.
