# Patrones y Buenas Prácticas

> [!definicion]
> Esta sección recoge los patrones y principios que hacen el código JavaScript más **mantenible**, **performante** y **predecible**. Cubre tanto técnicas de optimización puntual (debouncing, memoización) como principios arquitectónicos (separación de responsabilidades, código limpio) y patrones de diseño clásicos adaptados a JS moderno (Singleton, Observer). Son las herramientas que diferencian código que funciona de código que escala.
